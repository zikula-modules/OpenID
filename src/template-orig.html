Overview

Beginning with version 1.3.0, Zikula introduced it's Authentication API, which allows a module to provide authentication services in addition to those provided by the core Users module. A module providing authentication services allows a user to log into a Zikula site using credentials (something like, but not limited to, a user name and password) that are not managed by Zikula's core Users module. In addition, an authentication module can enable its authentication method(s) for registration, meaning that a new user can request a new web site account using those credentials instead of creating a User name and password.

The OpenID for Zikula module allows a user to use his OpenID as his credentials for both logging into, and for registering a new account with a Zikula-based web site.
